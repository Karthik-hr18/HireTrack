# USER_STORIES.md — HireTrack

23 stories total, across three roles. Every story traces to a product goal or a locked business rule.

---

## Recruiter (10)

| ID | Story |
|---|---|
| REC-01 | As a Recruiter, I want to create and manage job openings, so that candidates can apply for available positions. |
| REC-02 | As a Recruiter, I want to review candidate resumes, so that I can decide whether they should move to the next stage. |
| REC-03 | As a Recruiter, I want to add candidates manually with a required source (Referral, LinkedIn, Campus, etc.), so that all candidates enter the same hiring pipeline. |
| REC-04 | As a Recruiter, I want to view all candidates in a hiring pipeline grouped by stage, so that I can easily track recruitment progress. |
| REC-05 | As a Recruiter, I want to move candidates between hiring stages, so that the recruitment process stays updated. |
| REC-06 | As a Recruiter, I want to schedule interviews by assigning an Admin as the interviewer, so that interviews are organized within the system. |
| REC-07 | As a Recruiter, I want to search and filter candidates by name, job, stage, or source, so that I can quickly find any candidate. |
| REC-08 | As a Recruiter, I want to view a candidate's complete profile, including resume, notes, interviews, scorecards, and activity timeline, so that I have all the necessary information in one place. |
| REC-09 | As a Recruiter, I want to reject candidates during early hiring stages with a predefined rejection reason and optional notes, so that the pipeline remains accurate. |
| REC-10 | As a Recruiter, I want to view interview feedback submitted by the Admin, so that I can understand the interview outcome before proceeding. |

## Admin (7)

| ID | Story |
|---|---|
| ADM-01 | As an Admin, I want to view interviews assigned to me, so that I know which candidates I need to interview. |
| ADM-02 | As an Admin, I want to conduct interviews and submit interview scorecards with ratings and comments, so that every interview has structured feedback. |
| ADM-03 | As an Admin, I want to view a candidate's complete profile, including resume, notes, interviews, scorecards, and activity timeline, so that I can make informed hiring decisions. |
| ADM-04 | As an Admin, I want to move candidates to the next hiring stage after reviewing interview feedback, so that the hiring process progresses correctly. |
| ADM-05 | As an Admin, I want to make the final hiring decision by marking a candidate as Hired or Rejected, so that the recruitment process is completed. |
| ADM-06 | As an Admin, I want to view a hiring dashboard showing key recruitment metrics (open jobs, total candidates, candidates by stage, candidates by source), so that I can monitor the overall hiring process. |
| ADM-07 | As an Admin, I want to manage recruiter accounts, so that only authorized recruiters can access the system. |

## Candidate (6)

| ID | Story |
|---|---|
| CAND-01 | As a Candidate, I want to register an account, so that I can apply to jobs and track my applications over time. |
| CAND-02 | As a Candidate, I want to browse open job listings on the careers page, so that I can find roles that match my interests. |
| CAND-03 | As a Candidate, I want to view a job's full details (description, requirements, location), so that I understand the role before applying. |
| CAND-04 | As a Candidate, I want to apply to a job and upload my resume, so that my application enters the company's hiring pipeline. |
| CAND-05 | As a Candidate, I want to check my application status at any time, so that I know where I stand without emailing the recruiter. |
| CAND-06 | As a Candidate, I want to update my profile (contact info, resume), so that my details stay current for this and future applications. |
